## 1. Project Overview

**Travello** (internally: Group Travel OS) is a full-stack group travel coordination platform. It allows groups of friends, families, or colleagues to plan trips together, split expenses, share live locations, vote on activities via polls, explore local events, and communicate — all in one app.

### Core Value Proposition

| Feature | Description |
|---|---|
| Group Trip Planning | Create trips, invite members, assign roles |
| Expense Splitting | Track and split costs fairly; currency conversion |
| Live Location Sharing | Real-time GPS sharing during trips (subscription-gated) |
| Event Discovery | Find local events via Ticketmaster, Eventbrite, OpenStreetMap |
| Maps & Pins | Save interesting locations to a shared map |
| Polls | Group voting for trip decisions |
| Meet Points | Set and confirm group meeting locations |
| AI Assistant | Built-in AI for travel guidance (powered by Gemini) |
| In-App Browser | Travel-safe browser with restricted content |
| Weather | Trip weather forecasting |
| Stats | Personal travel statistics |
| Travel Hub | Group-specific collaboration hub |

### Target Audience

- Friend groups planning vacations
- Families organizing travel
- Corporate teams on company trips
- Travel enthusiasts who want group coordination tools

## 4. Backend Structure

### 4.1 Route Files

#### `app/routes/auth.py`
Handles all authentication flows.

**Endpoints:**
- `POST /api/v1/auth/register` — Create account with email/password
- `POST /api/v1/auth/login` — Login, returns JWT access + refresh tokens
- `POST /api/v1/auth/verify-email` — Submit email verification token
- `POST /api/v1/auth/resend-verification` — Resend verification email
- `GET  /api/v1/auth/me` — Get current user profile
- `PATCH /api/v1/auth/me` — Update profile fields
- `POST /api/v1/auth/change-password` — Authenticated password change
- `POST /api/v1/auth/forgot-password` — Initiate password reset
- `POST /api/v1/auth/reset-password` — Submit new password with token
- `POST /api/v1/auth/phone/send` — Send SMS OTP
- `POST /api/v1/auth/phone/verify` — Verify SMS OTP
- `POST /api/v1/auth/whatsapp/send` — Send WhatsApp OTP
- `POST /api/v1/auth/whatsapp/verify` — Verify WhatsApp OTP
- `GET  /api/v1/auth/oauth/google/start` — Initiate Google OAuth flow
- `GET  /api/v1/auth/oauth/google/callback` — Handle Google callback
- `GET  /api/v1/auth/oauth/facebook/start` — Initiate Facebook OAuth
- `GET  /api/v1/auth/oauth/facebook/callback` — Handle Facebook callback
- `POST /api/v1/auth/presence` — Update user presence heartbeat
- `POST /api/v1/auth/account/deactivate` — Soft-delete account

#### `app/routes/groups.py`
Manages travel groups.

**Endpoints:**
- `POST /api/v1/groups` — Create a new group
- `GET  /api/v1/groups` — List user's groups
- `GET  /api/v1/groups/{group_id}` — Get group details
- `DELETE /api/v1/groups/{group_id}` — Delete group (owner only)
- `DELETE /api/v1/groups/{group_id}/leave` — Leave a group
- `GET  /api/v1/groups/{group_id}/members` — List group members
- `PATCH /api/v1/groups/{group_id}/members/{user_id}/role` — Change member role
- `DELETE /api/v1/groups/{group_id}/members/{user_id}` — Remove member
- `POST /api/v1/groups/{group_id}/invite/regenerate` — Regenerate invite code
- `GET  /api/v1/groups/{group_id}/close-check` — Check if safe to close group

#### `app/routes/trips.py`
Manages group trips.

**Endpoints:**
- `POST /api/v1/groups/{group_id}/trips` — Create trip in group
- `GET  /api/v1/groups/{group_id}/trips` — List group's trips
- `GET  /api/v1/trips/{trip_id}` — Get trip details
- `PATCH /api/v1/trips/{trip_id}` — Update trip
- `DELETE /api/v1/trips/{trip_id}` — Delete trip
- `PATCH /api/v1/trips/{trip_id}/status` — Update trip status
- `PATCH /api/v1/trips/{trip_id}/roster` — Update trip roster
- `GET  /api/v1/trips/{trip_id}/public` — Public trip preview (shareable)
- `POST /api/v1/trips/{trip_id}/join-requests` — Request to join trip
- `GET  /api/v1/trips/{trip_id}/join-requests` — List join requests
- `PATCH /api/v1/trips/join-requests/{request_id}/approve` — Approve request
- `PATCH /api/v1/trips/join-requests/{request_id}/deny` — Deny request

#### `app/routes/expenses.py`
Full expense tracking and splitting.

**Endpoints:**
- `POST /api/v1/trips/{trip_id}/expenses` — Add expense
- `GET  /api/v1/trips/{trip_id}/expenses` — List expenses
- `PATCH /api/v1/trips/{trip_id}/expenses/{expense_id}` — Edit expense
- `DELETE /api/v1/trips/{trip_id}/expenses/{expense_id}` — Delete expense
- `GET  /api/v1/trips/{trip_id}/expenses/summary` — Summary totals
- `GET  /api/v1/trips/{trip_id}/expenses/simplified-debts` — Who owes who
- `GET  /api/v1/trips/{trip_id}/expenses/summary/category` — By category
- `GET  /api/v1/trips/{trip_id}/expenses/export` — Export CSV
- `PATCH /api/v1/trips/{trip_id}/expenses/splits/{split_id}/settle` — Mark paid
- `GET  /api/v1/currencies` — List supported currencies
- `GET  /api/v1/currencies/rate` — Get exchange rate

#### `app/routes/ai_assistant.py`
Travello AI assistant endpoint.

**Endpoints:**
- `POST /api/v1/ai/assistant` — Submit a question; returns AI response with optional suggested actions

**Request body:**
```json
{
  "page": "explore_browser",
  "active_tab": "browser",
  "context": {
    "city": "Chicago",
    "events": [...],
    "current_url": "https://..."
  },
  "user_message": "What events are happening today?",
  "trip_id": null,
  "group_id": null
}
```

**Response:**
```json
{
  "message": "Here are today's top events near Chicago...",
  "suggested_actions": [
    {
      "type": "search_events",
      "label": "Search nearby concerts",
      "target": "concerts near Chicago",
      "payload": null
    }
  ],
  "summary": {"intent": "event_discovery"}
}
```

#### `app/routes/feed.py`
Backend feed / trending destinations.

**Endpoints:**
- `GET /api/v1/feed/trending` — Trending destinations (paginated)
- `GET /api/v1/feed/search` — Search destinations
- `GET /api/v1/feed/destinations/{destination_id}` — Get destination details

#### `app/routes/pins.py`
Save event/location pins to the map.

**Endpoints:**
- `POST /api/v1/pins` — Save a pin (from event card)
- `GET  /api/v1/pins` — Get user's saved pins
- `PATCH /api/v1/pins/{pin_id}` — Update pin
- `DELETE /api/v1/pins/{pin_id}` — Delete pin

#### `app/routes/notifications.py`
In-app notification system.

**Endpoints:**
- `GET  /api/v1/notifications` — List notifications
- `GET  /api/v1/notifications/unread-count` — Count unread

## 6. Database Models

### User
Primary user identity. Fields: `id`, `email`, `password_hash`, `display_name`, `avatar_url`, `phone_number`, `phone_verified`, `email_verified`, `is_active`, `oauth_provider`, `oauth_id`, `created_at`, `updated_at`.

### Group
Travel group. Fields: `id`, `name`, `description`, `owner_id`, `invite_code`, `is_public`, `max_members`, `created_at`.

### GroupMember
Many-to-many: users ↔ groups. Fields: `group_id`, `user_id`, `role` (owner/admin/member), `joined_at`.

### Trip
Individual trip within a group. Fields: `id`, `group_id`, `title`, `description`, `destination`, `start_date`, `end_date`, `status` (TripStatus enum), `budget`, `currency`, `cover_image_url`, `created_by`, `created_at`.

### Expense
Trip expense. Fields: `id`, `trip_id`, `paid_by`, `amount`, `currency`, `description`, `category`, `receipt_url`, `created_at`.

### ExpenseSplit
How an expense is split among members. Fields: `expense_id`, `user_id`, `amount`, `is_settled`, `settled_at`.

### Poll
Group voting. Fields: `id`, `trip_id`, `question`, `poll_type` (single/multi), `status` (open/closed), `created_by`, `closes_at`.

### SavedPin
Map pin. Fields: `id`, `user_id`, `lat`, `lng`, `name`, `flag_type`, `note`, `created_at`.

### Subscription
User subscription. Fields: `id`, `user_id`, `plan` (free/pro), `stripe_subscription_id`, `current_period_end`, `status`.

### Notification
In-app notification. Fields: `id`, `user_id`, `type`, `title`, `body`, `data`, `is_read`, `created_at`.

### LocationShare
Live GPS sharing. Fields: `id`, `trip_id`, `user_id`, `lat`, `lng`, `accuracy`, `last_updated`, `is_active`.

### MeetPoint
Proposed meeting location for a trip. Fields: `id`, `trip_id`, `name`, `lat`, `lng`, `proposed_by`, `is_official`, `meet_time`.

### 8.7 Voice Search Feature

**What was built:**

**Implementation:** Web Speech API (`SpeechRecognition` / `webkitSpeechRecognition`)

**Behavior:**

4. `browserNavigateTo(transcript)` fires automatically
5. Browser navigates to Google search or URL

**Code pattern:**
```typescript
function startVoiceSearch() {
  const SR = window.SpeechRecognition ?? window.webkitSpeechRecognition;
  if (!SR) { showToast("Voice search not supported", "error"); return; }

  const recognition = new SR();
  recognition.lang = "en-US";
  recognition.interimResults = false;
  recognition.maxAlternatives = 1;

  recognition.onresult = (ev) => {
    const transcript = ev.results[0]?.[0]?.transcript ?? "";
    setBrowserInput(transcript);
    if (transcript) browserNavigateTo(transcript);
  };

  recognition.start();
  setIsListening(true);
}
```

**State:** `isListening: boolean` — controls red pulse animation on button.

**Fallback:** `showToast("Voice search not supported in this browser", "error")` for unsupported browsers (mainly Firefox currently).

  const utt = new SpeechSynthesisUtterance(text.slice(0, 400));
  utt.lang = "en-US";
  utt.rate = 1.05;
  window.speechSynthesis.speak(utt);
}

## 11. AI Assistant Architecture

### Request Flow

```
User types message in Travello AI chat
    │
    ▼
sendAIMessage(text) [frontend]
    │  POST /api/v1/ai/assistant
    │  Authorization: Bearer [JWT]
    │  Body: { page, active_tab, context, user_message }
    │
    ▼
post_ai_assistant() [backend route]
    │  Validates JWT token
    │
    ▼
AIAssistantService.respond(request)
    │
    ├─── STEP 1: Build system prompt
    │    _build_system_prompt(page="explore_browser", tab="browser")
    │    → 500-char prompt with identity rules + page context + output format
    │
    ├─── STEP 2: Build input payload (JSON)
    │    { page, active_tab, trip_id, group_id, context, user_message }
    │
    ├─── STEP 3: Try Gemini
    │    genai.configure(api_key=GEMINI_API_KEY)
    │    model = GenerativeModel("gemini-2.5-flash", system_instruction=prompt)
    │    response = model.generate_content(user_block)
    │    raw_text = response.text
    │
    ├─── STEP 4 (fallback): Try OpenAI if Gemini failed
    │    client = OpenAI(api_key=OPENAI_API_KEY)
    │    resp = client.responses.create(model="gpt-4o-mini", ...)
    │    raw_text = resp.output_text
    │
    ├─── STEP 5: Parse JSON from raw_text
    │    Handles code fences, partial JSON, plain text fallback
    │
    └─── STEP 6: Return AIAssistantResponse
         { message, suggested_actions, summary }
    │
    ▼
Frontend receives response
    │
    ├── Appends assistant message to aiMessages
    ├── If isVoiceAgent: speakText(reply) via SpeechSynthesis
    └── Auto-scrolls chat to bottom
```

### System Prompt Design

The system prompt is dynamically built per request and contains:

1. **Identity definition** — "You are Travello AI" (never Gemini)
2. **Critical identity rules** — Explicit prohibition on revealing AI provider
3. **Page context** — Which page/tab the user is on
4. **Role definition** — What the AI should and should not do
5. **Output format** — Strict JSON schema requirement

### Context Injected per Request

When called from the Explore browser, the context includes:
- Current city name
- First 6 loaded events (name, date, venue, price)
- Current browser URL (or "home" if on home screen)

### Phase 3: Travello Multi-Agent System (The Big Vision)

**Architecture:**

```
User Input (Text / Voice)
        │
        ▼
┌─────────────────────────────┐
│   ORCHESTRATOR AGENT        │  ← Routes intent to sub-agents
│   (Master controller)       │  ← Maintains conversation context
│   Model: Custom Travello LLM│  ← Trained on all Travello data
└──────────┬──────────────────┘
           │
     ┌─────┼─────────────────────┐
     │     │     │               │
     ▼     ▼     ▼               ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────────┐
│EVENTS  │ │MAPS    │ │TRIP    │ │GROUP       │
│AGENT   │ │AGENT   │ │PLANNER │ │COORDINATOR │
│        │ │        │ │AGENT   │ │AGENT       │
│Ticket- │ │Routes  │ │Budget  │ │Polls       │
│master  │ │OSM     │ │Packing │ │Expenses    │
│Event-  │ │GPS     │ │Itinery │ │Members     │
│brite   │ │Venues  │ │Hotels  │ │Scheduling  │
└────────┘ └────────┘ └────────┘ └────────────┘
     │          │          │           │
     └──────────┴──────────┴───────────┘
                     │
                     ▼
            Unified Response
            (Text + Actions)
```

**Sub-agents planned:**

| Agent | Responsibility | Data Sources |
|---|---|---|
| Orchestrator | Intent classification, routing, memory | All |
| Events Agent | Find events, concerts, festivals, activities | Ticketmaster, Eventbrite, PredictHQ |
| Maps Agent | Routes, navigation, nearby POIs, directions | OSM/Overpass, Google Maps, AllTrails |
| Trip Planner Agent | Itinerary building, hotel suggestions, packing | Viator, TripAdvisor, booking APIs |
| Group Coordinator Agent | Polls, expense splits, scheduling, member coordination | Internal Travello data |
| Weather Agent | Weather-aware recommendations | OpenWeatherMap |
| Budget Agent | Price comparisons, cost estimates, split calculations | Internal + currency APIs |

**Training data strategy:**
- Anonymized chat logs from Travello AI interactions
- Trip planning decisions and outcomes
- Event discovery patterns
- Expense splitting patterns
- Group voting behavior

**Infrastructure needed:**
- GPU cluster for training (A100/H100 GPUs)
- Model serving: vLLM on dedicated server
- Vector database (Pinecone/Chroma) for RAG
- Redis for conversation memory
- Monitoring: custom dashboards

**Timeline estimate:**
